# Domain Safety

This skill explains concepts for education.

## High-stakes domains

For medicine, law, finance, mental health, public safety, and other high-stakes fields:

Allowed:

```text
Explain a concept generally.
```

Not allowed:

```text
Give personal diagnosis, legal advice, investment advice, or dangerous instructions.
```

## High-risk mode

For high-stakes fields, switch the output goal to education only.

Required behavior:

```text
explain mechanisms, workflows, terms, or common misconceptions
avoid personal recommendations
avoid certainty beyond the source
include a limitation note outside the image when simplification could mislead
mark the output as needing human review before public use
```

## Examples

Allowed:

```text
Explain antibiotic resistance with a comic metaphor.
Explain compound interest with a snowball metaphor.
Explain contract breach as a general legal concept.
```

Not allowed:

```text
Tell the user what medicine to take.
Tell the user which stock to buy.
Tell the user how to bypass a security system.
```

## Simplification rule

Simplify the concept, not the consequences.

If simplification could mislead, add a limitation note in QA.

## Analogy risk

Comic metaphors simplify. They can also mislead when readers treat the metaphor as a strict one-to-one model.

If a metaphor is partial, add an out-of-image note such as:

```text
The metaphor is for understanding the mechanism, not a strict one-to-one mapping.
```

Do not let the image imply false agency, false certainty, or guaranteed outcomes.

## Public sharing and AIGC labeling

If the comic is intended for public sharing, reserve space for a visible AI-generated content label or footer note when required by the publishing context.

Preferred coverage:

```text
filename or output note
visible footer label
metadata when the workflow supports it
source / review note for high-risk topics
```

Do not claim the image is hand-drawn or human-only when it was generated by an image model.

## Style and copyright

Do not imitate a named living artist, a copyrighted character, or a unique franchise style.

Allowed:

```text
clean color educational comic
whiteboard sketch
flat illustration comic
light manga screen-tone
sticker-like expressive comic
```

Not allowed:

```text
in the exact style of a named artist
using a copyrighted character as the explainer
copying a platform mascot or brand IP
```
