# Example: RAG vs Knowledge Graph QA

## Input

```text
把 RAG 和知识图谱问答的区别做成漫画
```

## Output

概念：RAG vs 知识图谱问答

一句话解释：
RAG 更像从文本里找相关证据，知识图谱问答更像沿着实体关系寻找答案路径。

概念类型：
Comparison Concept

漫画隐喻：
左边是助手在资料堆里找相关纸片；右边是侦探沿着关系线索图追踪实体路径。

推荐版式：
左右对比型

分镜脚本：
左侧：普通 RAG 在纸堆中拿出几张“相关段落”，旁边有“Top-K”“相似度”标签。
右侧：知识图谱问答站在关系白板前，沿着“实体A→关系→实体B→来源”找答案。
底部：RAG 擅长找证据，图谱擅长理关系；两者也可以结合。

图中文字：
标题：找段落，还是找关系？
标签：相关段落、实体、关系、来源、路径
总结：RAG 找证据，图谱理关系。

QA：
不要把 RAG 画成完全没用；对比要准确，不要贬低一方。
